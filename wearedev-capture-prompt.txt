**SHARED MIC COMPONENT — build once, use in Sessions + Contacts**

Build a reusable `<MicCapture>` component with these props:
- `onTranscript(structuredData)` — callback with parsed JSON from Claude
- `parsePrompt` — the system prompt string to use for this context (sessions vs contacts)
- `label` — button label e.g. "Record Session Note" or "Record Voice Note"

Behavior:
1. Button shows mic icon + label. Tap → requests `getUserMedia({ audio: true })`
2. If mic permission denied: show inline muted text "Mic unavailable — fill in manually". Do not crash. Do not show error modal.
3. While recording: button turns red `#ef4444`, pulses, shows "Recording... tap to stop"
4. On stop: show "Transcribing..." loading state
5. Encode audio blob as base64
6. Send to Anthropic API:
```js
{
  model: "claude-sonnet-4-6",
  max_tokens: 1000,
  messages: [{
    role: "user",
    content: [
      {
        type: "document",
        source: {
          type: "base64",
          media_type: audioBlob.type, // e.g. "audio/webm"
          data: base64AudioData
        }
      },
      {
        type: "text",
        text: parsePrompt
      }
    ]
  }]
}
```
7. Parse JSON response → call `onTranscript(parsedData)` → parent pre-fills form fields
8. If JSON parse fails: show raw transcript text so user can read and fill manually
9. After transcription: show small "Re-record" link to try again

---

**TAB 1 — SCAN (Business Card Scanner)**

UI:
- "SCAN CARD" header
- Large camera viewfinder area — uses `getUserMedia({ video: true })` to stream device camera
- "Capture" button (circular, purple `#8b5cf6`) overlaid at bottom of viewfinder
- Fallback: "Upload photo instead" text link — file input accept="image/*" capture="environment"
- After capture/upload: show thumbnail + "Parsing card..." loading state

Behavior:
- Convert image to base64, send to Anthropic API:
```
System: You are a business card parser. Extract the following fields from the business card image. Return ONLY a JSON object with no preamble, no markdown, no backticks. Fields: name (string), title (string), company (string), email (string), linkedin (string, if visible), phone (string, if visible). If a field is not visible, return an empty string for that field.
```
- Parse JSON → pre-fill editable form:
  - Name, Title, Company, Email, LinkedIn (optional), Phone (optional), Notes (free text)
- "Save Contact" button (teal) → saves to contacts state
- "Scan Another" resets view
- If JSON parse fails: show raw API response as text, all fields blank for manual entry

**TAB 2 — SESSIONS (Workshop/Talk Logger)**

UI:
- "SESSIONS" header
- "Log Session" button (orange) at top right
- Session list, reverse chronological, below

Log Session — inline expanded form (not a modal):

At the TOP of the form, before any text fields:

```
<MicCapture
  label="Record Session Note"
  parsePrompt={SESSION_PARSE_PROMPT}
  onTranscript={(data) => pre-fill all session fields from data}
/>
```

SESSION_PARSE_PROMPT:
```
The user has recorded a voice note about a conference session they just attended. Extract and return ONLY a JSON object with no preamble, no markdown, no backticks. Fields: sessionTitle (string), speaker (string), day (string — one of: "Wed", "Thu", "Fri", or empty string), timeSlot (string), keyInsight (string — the most useful thing they mentioned), quoteStat (string — any quote or statistic mentioned), actionItem (string — anything they said they'd do next), rating (number 1-5, infer from sentiment if not stated explicitly, default 3).
```

Text fields below the mic (pre-filled by voice, editable):
- Session title
- Speaker name
- Day (segmented tap: Wed / Thu / Fri)
- Time slot
- Key insight (textarea)
- Quote or stat (textarea)
- Action item (textarea)
- Rating (1–5 star tap selector, purple)
- "Save Session" button (teal)

Session card: title, speaker, day+time, stars, truncated insight. Tap to expand.
